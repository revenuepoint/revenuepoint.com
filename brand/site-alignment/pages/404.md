# 404 / not-found — *missing route*

> Source: *no `not-found.tsx` exists in `src/app/`*. Default Next.js 404 surface renders without RevenuePoint copy.

## Summary

The site has **no custom 404 page**. Hitting an unknown URL returns the default Next.js *404 — Page not found.* — which leaks the framework's voice instead of the brand's. This is an IA/UX gap, not a voice rewrite — but it should ship before launch alongside the rest of the polish.

This file documents the recommended copy for the new `src/app/not-found.tsx`. Implementation is part of Phase 2.

---

## Recommended copy (new file)

**Location:** new `src/app/not-found.tsx`

**Proposed (full file):**

```tsx
import Link from 'next/link';
import { buildMetadata } from '@/lib/metadata';
import { Button } from '@/components/ui/Button';

export const metadata = buildMetadata({
  title: 'Page not found',
  description: 'The page you were looking for does not exist on revenuepoint.com.',
  path: '/404/',
});

export default function NotFound() {
  return (
    <section className="bg-paper py-24 lg:py-32">
      <div className="max-w-2xl mx-auto px-6 lg:px-8 text-center">
        <p className="font-mono text-[11px] uppercase tracking-[0.16em] text-mute justify-center inline-flex items-center gap-2">
          <span className="h-px w-8 bg-rust" /> 404 · page not found
        </p>
        <h1 className="mt-6 text-d1 font-serif font-semibold text-ink">
          That page <em>does not exist</em>.
        </h1>
        <p className="mt-6 text-lede text-inkSoft leading-[1.65] max-w-prose mx-auto">
          The link is wrong, the page moved, or it never existed. Try one of these instead.
        </p>
        <div className="mt-10 flex flex-wrap gap-4 justify-center">
          <Button variant="primary" href="/">Go to homepage</Button>
          <Button variant="secondary" href="/foundry/">Explore Foundry</Button>
          <Button variant="secondary" href="/contact/">Get in touch</Button>
        </div>
      </div>
    </section>
  );
}
```

**Rationale:** Mirrors the `thank-you` page's structure (status pill at top, italicized-noun h1, lede body, CTA stack) so the brand voice carries across the system surfaces. The italicized *does not exist* sits in the same headline mechanic the rest of the site uses (`editorial-style.md` §three-beat headline mechanics). Three CTAs cover the most common buyer recovery paths: homepage, the flagship product, the contact form. The *404 · page not found* eyebrow uses the rust color (signal-off-track) per the brand's color tokens.

**Priority:** P1 — IA gap; ship before launch but not blocking other voice work.

## Out of scope, flagged

- **Multilingual or geo-routed 404s** (none planned).
- **Custom 410 / 451 / etc. error pages** — Next.js handles via `error.tsx` boundaries; if we want branded versions of those, separate workstream after launch.
- **Sitemap / `robots.txt` review** — out of scope for voice work; flag for SEO follow-up.
